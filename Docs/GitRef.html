Git Reference

Following is a quick reference providing a categorized overview of the most commonly used git commands.

Git is a distributed version control system designed to manage and track changes in source code. It allows developers to collaborate efficiently while maintaining a complete history of changes.

Git helps developers manage project versions, track changes, and maintain a history of all modifications. Even when working individually, it allows developers to manage different versions of their code, experiment with new features safely, and revert to previous states if needed. Its support for branching, merging, and distributed workflows simplifies development tasks, while maintaining a clear history of changes for future debugging and enhancements.

Git is widely used across industry, academia, and open-source projects. Companies rely on it for software development, and open-source contributors worldwide use it to collaborate on projects of all sizes.

Git commands are categorized based on functionality, including repository setup, file tracking, branch management, remote collaboration, history inspection, and undoing changes.

1. Repository Setup and Configuration

Command Description Options Usage Example
git init Initializes a new Git repository None git init
git clone Copies a repository from a remote source -b (branch), --depth (shallow clone) git clone URL
git config Sets user configuration details --global (all projects) git config --global user.name "John"

2. File and Change Management

Command Description Options Usage Example
git add Stages changes for commit -A (all files), . (current directory) git add filename
git commit Records changes with a commit message -m (commit message), --amend (edit last commit) git commit -m "message"
git rm Removes files from the repository --cached (unstage file), -r (recursive) git rm filename
git mv Renames or moves files None git mv old.txt new.txt

3. Branching and Merging

Command Description Options Usage Example
git branch Manages branches -a (list all branches), -d (delete branch) git branch new-branch
git checkout Switches branches or restores files -b (create new branch) git checkout main
git switch Switches branches (alternative to checkout) None git switch new-branch
git merge Combines changes from different branches --no-ff (disable fast-forward merging) git merge main
git rebase Reapplies commits on a new base branch --continue (after conflict resolution) git rebase main

4. Remote Repository Management

Command Description Options Usage Example
git remote Manages remote repositories -v (show remote URLs) git remote add origin URL
git fetch Retrieves changes from a remote repository --all (all branches) git fetch origin
git pull Fetches and merges changes from a remote repository --rebase (apply commits without merge) git pull origin main
git push Uploads changes to a remote repository -u (set upstream), --force (overwrite changes) git push origin main

5. Inspection and Comparison

Command Description Options Usage Example
git status Displays current repository status None git status
git log Displays commit history --oneline (compact view), --graph (visual) git log --oneline
git show Shows details of a specific commit None git show HEAD
git diff Shows changes between commits or files --cached (staged changes) git diff
git blame Shows who made changes to each line of a file -L (limit to specific lines) git blame filename

6. Undoing Changes and History Rewriting

Command Description Options Usage Example
git reset Resets changes in the working directory --soft (keep changes), --hard (discard changes) git reset HEAD~1
git revert Creates a new commit to undo changes None git revert HEAD
git stash Temporarily saves uncommitted changes pop (restore changes), list (view stashes) git stash
git cherry-pick Applies specific commits from another branch None git cherry-pick <commit-hash>
git restore Restores working directory files --staged (restore from index) git restore filename
