Windows Subsystem for Linux (WSL)

Introduction

Windows Subsystem for Linux (WSL) allows users to run a Linux environment directly on Windows, providing seamless integration between the two systems. It is designed for developers and power users who need Linux tools and commands while working in a Windows environment.

WSL has two versions: WSL1 and WSL2. WSL2 is the newer, faster version that uses a lightweight virtual machine (VM) architecture running on a hypervisor. A hypervisor is a software layer that allows multiple operating systems to share the same physical hardware. It provides virtualization, which means the Linux kernel runs in a VM but integrates so seamlessly that users experience it as part of Windows.

Unlike traditional virtual machines, WSL2 is optimized for performance and usability, so it feels like native Linux without the need for dual-boot setups. The main advantage is that you can switch between Windows and Linux tools instantly, with shared file access.

WSL2 is available on the following Windows versions:

  • Windows 10 Version 1903 (Build 18362) or later (Pro, Enterprise, or Education editions).
  • Windows 11 (all editions).

Why Do We Need WSL for this Course?

This course explores operating systems concepts, such as process management, threading, memory management, and scheduling algorithms, which are deeply tied to Unix-based systems. Unix environments like Linux provide the tools and architecture necessary to interact directly with these underlying concepts.

Linux is a good and widely used choice because it is open-source, powerful, and versatile. Its system design makes it ideal for learning core operating systems concepts through direct, hands-on experience. Additionally, Linux is heavily used in the software development industry, cloud environments, and server management, making it an essential skill for most engineers.

Installing WSL and Ubuntu

To use WSL, you first need to install and set up the environment on your Windows system. Below are the steps to ensure everything is ready.

Check System Requirements

Before installing WSL, verify that your system meets the requirements. This step ensures compatibility and avoids issues during installation.

Note: Virtualization must also be enabled in your BIOS (this is enabled by default in most modern PCs; therefore, it is not explained below. If, however, you run into this issue, please let me know).

Enable WSL

Enabling WSL sets up the Linux subsystem on your Windows machine. This command automates the installation process.

wsl --install

This command installs WSL, sets WSL 2 as the default version, and installs the default Linux distribution (e.g., Ubuntu).

Complete Linux Distribution Setup

Once WSL is installed, you need to configure the Linux distribution. This includes setting up your username and password for the Ubuntu environment. When prompted for a username and password, use a simple username (such as your first name) and a short and memorable password.

Launch Ubuntu, follow the on-screen prompts, and complete the initial configuration.

ubuntu

You are now in a Bash terminal using the Bash shell.

Access Files

After installation, you can access your files from both Windows and Linux environments. The file paths differ depending on the operating system.

Windows files are accessible in WSL under /mnt/drive_letter, such as /mnt/c. Conversely, Linux files are accessible from Windows using \\wsl$\ in the file explorer's address bar.

File paths in Linux

In Linux, file paths describe the location of files and directories within the file system. Paths can be absolute, starting from the root directory (/), or relative, starting from the current working directory. For example, the absolute path /home/user/Documents/file.txt specifies the exact location of file.txt, whereas the relative path Documents/file.txt refers to the same file when your current directory is /home/user.

Linux uses a forward slash (/) to separate directories in a path. Special symbols like . (current directory) and .. (parent directory) help navigate the file system. For instance, if you are in /home/user and want to access file.txt in Documents, the relative path would be ./Documents/file.txt, and moving up one level to /home would require cd ... Understanding these basics makes working with the Linux file system more efficient.

Installing g++ and lldb on Ubuntu

To compile and debug C++ code, as you will be doing in this course, you need to install tools like g++ and lldb in your Ubuntu environment. (You need to install g++ in Ubuntu even if you already have it in the Windows environment since we will be using it from within Ubuntu.) These tools are standard for Linux-based development. Note: I will be introducing lldb in more detail on a later page.

sudo apt update
sudo apt install g++ lldb

apt is the package manager for Linux, used to install, update, and manage software. sudo grants administrative privileges needed for system-level tasks. The command sudo apt update refreshes the list of available software, and sudo apt install g++ lldb installs the GNU C++ compiler and the LLVM Debugger for compiling and debugging C++ programs.

Installing the WSL Extension on VSCode

To integrate VSCode with WSL, you need to install the 'WSL' extension, from the VSCode extensions pane. This extension enables a seamless workflow between your Windows editor and the Linux environment.

Install the Extension

Open Visual Studio Code from Windows, navigate to the Extensions Marketplace, search for 'WSL' extension by Microsoft, and click Install.

Open WSL in VS Code

Once the extension is installed, you can open a new WSL session directly in VS Code. This connects your editor to the Linux environment.

Press Ctrl + Shift + P to open the Command Palette. Type WSL: New Window and select it to open a WSL session in VS Code.

Open a Linux Folder

To work on files within the Linux environment, open a folder from WSL in VS Code. This allows you to edit and debug code seamlessly.

Use the menu option File > Open Folder in VS Code to navigate to your WSL file system. A common path to open is /home/username.

Using VS Code with WSL

Once connected to WSL, you can use VS Code to edit, debug, and run Linux-based projects seamlessly. Extensions like C++ tools can be installed inside WSL for enhanced integration.

Compiling a Simple C++ Program

Follow these steps to create and compile a simple C++ program (e.g., helloworld.cpp) from WSL within VS Code:

1. Open a new terminal within VSCode. In the terminal, create a folder named hello_world to organize your files:

mkdir hello_world

Navigate into the folder using:

cd hello_world

2. Open VS Code connected to WSL, and inside the hello_world folder, create a file named helloworld.cpp. Save the file by pressing Ctrl + S. If you encounter issues saving the file, see the section below on file permissions.

3. Compile the program using the following command:

g++ helloworld.cpp -o helloworld

This creates an executable file named helloworld.

4. Run the program with the command:

./helloworld

You should see the expected output of your program.

Handling File Permissions

If you encounter issues saving or compiling the file due to insufficient permissions, you may need to update the file's ownership. Run the following command in the terminal to assign ownership of the file to your user:

sudo chown your_username:your_username helloworld.cpp

chown stands for change ownership; it modifies the ownership of a file or directory, assigning it to a specified user and/or group.

Replace your_username with your WSL (Ubuntu) username. After this, you should be able to save and compile the file without any issues.
